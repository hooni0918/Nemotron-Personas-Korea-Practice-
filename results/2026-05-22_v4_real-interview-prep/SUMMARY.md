# v4 — 실사용자 인터뷰 준비 도구셋 (2026-05-22)

> v4는 **시뮬 인터뷰를 새로 돌리지 않습니다.** 대신 **실제 사용자 인터뷰가 곧 진행될 예정**이라, 그 인터뷰 결과를 v3 시뮬과 비교 분석할 수 있도록 **인터뷰 가이드 + 분석 템플릿 + 모집 가이드** 3종을 미리 준비.

---

## 🎯 왜 시뮬을 새로 안 돌리나

v1~v3로 핵심 가설은 이미 확인됨:
- ✅ 자동 저장 가치 (87% 만장일치)
- ✅ 재탐색 페인 보편성 (75%)
- ✅ 마케터 그룹 기능 선호 (40% vs 일반 20%)

여기서 시뮬을 더 돌리면 같은 답을 더 많이 듣는 것. **한계 효용 0에 수렴**.

**진짜 가치 있는 다음 단계**:
> 시뮬과 실제 사용자 응답을 짝지어 **시뮬 자체의 신뢰도를 검증** → 어떤 시뮬 결과를 디자이너·기획자에게 자신 있게 인용할 수 있는지 명확하게.

이 비교를 깔끔하게 하려면 인터뷰 가이드부터 v3 시뮬 질문에 매핑되게 짜야 함. **그래서 v4 = 인터뷰 도구셋**.

---

## 📦 산출물 3종

### 1️⃣ [recruiting_guide.md](./recruiting_guide.md)

5~8명 모집 가이드. 시뮬과 동일한 라벨 비율 (A_marketer 2~3 · B_engineer 1~2 · C_general 2~3).

핵심:
- **스크리닝 질문 5개** — 인스타 활발 사용 + 최근 저장 경험 있는 사람만 통과 (시뮬과 apples-to-apples 보장)
- **모집 채널 후보** — 디스코드·뉴스레터·지인·모집 서비스
- **사례비 가이드** (1~5만원)
- **모집 LP 메시지 템플릿**

### 2️⃣ [interview_guide_v1.md](./interview_guide_v1.md)

60~75분 1대1 인터뷰 스크립트. 6 섹션, Q1-Q20 의도 유지하되 자연스러운 대화 흐름.

**시간 배분의 핵심**: 시뮬 시그널 강도와 반대로 배분

| 섹션 | 시간 | 시뮬에서의 상태 |
|---|---|---|
| 워밍업 + 일상 행동 | 10분 | Q1, Q2 빠르게 |
| 페인 깊게 파기 | 15분 | Q3~Q8 (시뮬에서 강한 시그널, 인터뷰는 정성 보강) |
| **현재 무료 콤보 대체 (Q19/Q20)** | **15분** | 시뮬에서 카운트 거짓 양성, **인터뷰만 가능** |
| **가격 검증** | **10분** | 시뮬 데이터 **없음** |
| **친구 초대 진입** | **10분** | 시뮬 1인칭이라 **불가능** |
| MVP1 빠른 확인 (Q13/15/16) | 10분 | 시뮬에서 이미 강함, 빠른 검증만 |

→ **시뮬에서 안 풀린 영역에 인터뷰 시간 절반 이상 투자**. 핵심 원칙: "왜?" 3번 묻기, 객관식 보기 읽지 말 것, 우리 앱 이름·기능 인터뷰 후반까지 언급 X.

### 3️⃣ [analysis_template.md](./analysis_template.md)

인터뷰 끝난 직후 채울 빈칸 템플릿.

**구조**: 각 질문마다 4칸
- 시뮬 결과 (이미 채워져 있음)
- 실제 응답 (P1~P8 빈칸)
- 🟢🟡🔴 라벨
- **디자이너·기획자 take-away** ← 핵심

채우면 그대로 v5 SUMMARY가 됨. 마지막에 **의사결정 매트릭스** (와이어프레임 우선순위·LP 헤드라인·베타 GTM)로 정리.

---

## 🏷️ 핵심 라벨링 시스템

| 라벨 | 의미 | 디자이너·기획자에게 |
|---|---|---|
| 🟢 일치 | 시뮬 분포와 실제 응답 패턴이 거의 같음 | "**시뮬 결과를 정량 근거로 자신 있게 인용 가능**" |
| 🟡 부분 일치 | 큰 방향은 같지만 비율·강도 다름 | "시뮬은 참고만, 실제 응답 우선" |
| 🔴 다름 | 시뮬과 실제가 반대 방향 또는 완전히 다름 | "**시뮬 결과 폐기, 실제 인터뷰로만 의사결정**" |

→ 인터뷰 후 이 라벨이 붙은 **시뮬 신뢰도 매트릭스**가 v4의 최종 산출물.

## 🎁 v5에서 디자이너·기획자가 받게 될 산출물

1. **🟢 라벨 모음** — "이 시뮬 결과는 실제와 일치하니 LP·온보딩에 자신 있게 인용 OK"
2. **🔴 라벨 모음** — "이건 시뮬을 폐기하고 실제 인터뷰 멘트만 인용"
3. **시뮬에 없는 영역 정성 데이터** — 가격·진짜 전환·친구 초대 진입 (인터뷰만 줄 수 있는 것)
4. **와이어프레임 우선순위** A/B/C — 이번 스프린트, 다음 스프린트, 보류
5. **LP 헤드라인 후보** — 마케터가 그대로 쓸 수 있는 카피
6. **베타 GTM 전략** — 1순위 타겟·가격 정책·진입 화면 결정

# v4 — 실사용자 인터뷰 준비 도구셋 (2026-05-22)

## 한 줄 요약

v4에서는 시뮬 인터뷰를 새로 돌리지 않습니다. 실제 사용자 인터뷰가 곧 진행될 예정이라, 그 인터뷰 결과를 v3 시뮬과 비교 분석할 수 있도록 **인터뷰 가이드 + 분석 템플릿 + 모집 가이드** 3종을 미리 준비했습니다.

---

## 왜 시뮬을 새로 안 돌리나

v1~v3로 핵심 가설은 이미 확인했습니다.

- 자동 저장 가치 — 87% 만장일치 ✅
- 재탐색 페인의 보편성 — 75% ✅
- 마케터의 그룹 기능 선호 — 40% vs 일반 20% ✅

여기서 시뮬을 더 돌리면 같은 답을 더 많이 듣는 셈입니다. 한계 효용이 빠르게 0에 가까워져요.

진짜 가치 있는 다음 단계는 시뮬과 실제 사용자 응답을 짝지어 **시뮬 자체의 신뢰도를 검증**하는 것입니다. 어떤 시뮬 결과를 디자이너·기획자에게 자신 있게 인용할 수 있는지 명확하게 라벨링하는 게 목표예요.

이 비교를 깔끔하게 하려면 인터뷰 가이드부터 시뮬 질문에 매핑되게 짜야 합니다. 그래서 v4 = 인터뷰 진행 도구셋이 됩니다.

---

## 만든 산출물 4개

### 1. [`recruiting_guide.md`](./recruiting_guide.md) — 5~8명 모집

시뮬과 동일한 라벨 비율(A_marketer 2~3명 · B_engineer 1~2명 · C_general 2~3명).

핵심:
- 스크리닝 질문 5개 — 인스타 활발 사용 + 최근 저장 경험 있는 사람만 통과 (시뮬과 비교 가능하도록)
- 모집 채널 후보 — 디스코드·뉴스레터·지인·모집 서비스
- 사례비 가이드 (1~5만원)
- 모집 LP·메시지 템플릿

### 2. [`interview_guide_v1.md`](./interview_guide_v1.md) — 60~75분 인터뷰 스크립트

6개 섹션으로 나뉘어 있고, Q1-Q20 의도는 유지하되 자연스러운 대화 흐름으로 재구성했습니다.

**시간 배분에서 핵심**: 시뮬 시그널 강도와 반대로 배분했습니다.

| 섹션 | 시간 | 왜 이 시간? |
|---|---|---|
| 워밍업 + 일상 행동 | 10분 | Q1, Q2는 시뮬에서 이미 강한 신호 — 빠르게 |
| 페인 깊게 파기 | 15분 | Q3~Q8 시뮬은 양, 인터뷰는 정성 보강 |
| **현재 무료 방식 대체 (Q19/Q20)** | **15분** | 시뮬 카운트 거짓 양성, 인터뷰만 가능 |
| **가격 검증** | **10분** | 시뮬 데이터 자체가 없음 |
| **친구 초대 진입** | **10분** | 시뮬 1인칭이라 불가능 |
| MVP1 빠른 확인 (Q13/15/16) | 10분 | 시뮬에서 이미 강함 — 검증만 |

시뮬에서 못 푼 영역에 인터뷰 시간 절반 이상을 투자합니다.

핵심 원칙: "왜?"를 세 번 묻기, 객관식 보기는 그대로 읽지 말기, 우리 앱 이름이나 기능은 인터뷰 마지막 섹션까지 절대 언급하지 말기.

### 3. [`analysis_template.md`](./analysis_template.md) — 인터뷰 후 빈칸 채우기

각 질문마다 칸 4개:
- 시뮬 결과 (이미 채워져 있음)
- 실제 응답 (P1~P8 빈칸)
- 🟢🟡🔴 라벨
- 디자이너·기획자 take-away ← 핵심

다 채우면 그대로 v5 SUMMARY가 됩니다. 마지막에 **의사결정 매트릭스** (와이어프레임 우선순위·LP 헤드라인 후보·베타 GTM)로 정리됩니다.

## 핵심 라벨링 시스템

| 라벨 | 어떤 경우인가 | 디자이너·기획자에게 |
|---|---|---|
| 🟢 일치 | 시뮬 분포와 실제 응답 패턴이 거의 같음 | 시뮬 결과를 정량 근거로 자신 있게 인용해도 됩니다 |
| 🟡 부분 일치 | 큰 방향은 같지만 비율·강도가 다름 | 시뮬은 참고만 하고, 결정은 실제 응답으로 |
| 🔴 다름 | 시뮬과 실제가 반대 방향이거나 완전히 다름 | 시뮬 결과 폐기, 실제 인터뷰만 사용 |

인터뷰 후에 이 라벨이 붙은 **시뮬 신뢰도 매트릭스**가 v4의 최종 산출물이 됩니다.

## v5에서 디자이너·기획자가 받게 될 산출물

1. **🟢 라벨 모음** — 시뮬 결과를 LP·온보딩에 자신 있게 인용 가능
2. **🔴 라벨 모음** — 시뮬 폐기, 실제 인터뷰 멘트만 인용
3. **시뮬에 없는 영역 정성 데이터** — 가격·진짜 전환·친구 초대 진입
4. **와이어프레임 우선순위** A/B/C — 이번 스프린트, 다음 스프린트, 보류
5. **LP 헤드라인 후보** — 마케터가 그대로 쓸 수 있는 카피
6. **베타 GTM 전략** — 1순위 타겟·가격 정책·진입 화면 결정

> 2026-05-18 ~ 2026-05-20 사이에 진행한 세 번의 인터뷰 시뮬에서 **반복적으로 확인된 패턴**과 **회차 사이에 달라진 점**을 한 페이지로 모았습니다.

## ✅ 세 번 모두 일관되게 확인된 것

### 1. 핵심 페인 — "저장은 쉬운데 다시 못 찾는다"

저장 후 재탐색의 어려움이 모든 회차·모든 직군에서 반복 확인.

| 회차 | 페르소나 | 페인 호소 비율 |
|---|---|---|
| v1 | Nemotron 랜덤 50명 | Q3 "저장하고 까먹음"이 가장 빈번한 안 가는 이유 |
| v2 | 사용자 설계 3 타겟 | 수집형 5/5 "자주" + 실행형 3/5 "자주" |
| v3 | Nemotron 매칭 120명 | **64%가 "자주" / 75%가 "다시 찾기 매우 자주·자주"** |

→ **이 페인이 서비스가 풀어야 할 가장 명확한 문제**. 가설이 아니라 결론으로 굳어졌음.

### 2. MVP1 "링크 자동 저장" 가치 명제

| 회차 | 검증 강도 |
|---|---|
| v1 | Q5에서 "검색 안 해도 됨"이 가장 매력적 장점으로 언급 (Positive 56%) |
| v2 | 저장형 페르소나 4/5 "매우 편할" (즉흥형도 거부 X) |
| v3 | **120명 중 105명(87.5%) "매우 편할"** |

→ **만들어도 된다는 신호**가 점점 더 강해짐. 핵심 가치 검증 완료.

### 3. 친구·그룹 기능은 마케팅·콘텐츠 직군이 가장 강한 반응

v2에서 처음 발견된 "그룹 기능은 실행형(브랜드 마케터) > 수집형(마케팅 AE)" 패턴이 v3 120명 표본에서도 그대로 재현:

| 회차 | 직군 | "매우 그렇다" 비율 |
|---|---|---|
| v2 | 실행형 (브랜드 마케터) | 5/5 |
| v3 | A_marketer (40명) | **40%** |
| v3 | B_engineer (30명) | 33% |
| v3 | C_general (50명) | 20% |

→ **베타 모집 1순위 = 마케팅·콘텐츠·디자인 직군** 결정.

## ⚠️ 회차마다 새롭게 드러난 한계

### v1에서 드러난 것 — 진짜 경쟁자는 다른 앱이 아니다

> "지금도 스크린샷 찍거나 메모 앱에 적어두는 걸로 충분히 관리하고 있는데, 굳이 매달 2,900원씩 돈을 내면서까지 써야 할지는 잘 모르겠어요."

가격 거절 사유에서 발견된 가장 의미 있는 한 줄. **경쟁자는 다른 장소 저장 앱이 아니라 인스타 저장함 + 카톡 나에게 보내기 + 메모장 + 스크린샷의 무료 워크플로우**. 차별화 메시지는 "지도 뷰"만으로는 약하다는 것.

### v2에서 드러난 것 — Q19 "전환 의향"은 신중하게 봐야

다수가 "일부 상황에서만 사용" 응답. **완전 대체는 약함**. 이는 단순히 답변 톤만 보면 그렇지만, 자동 sentiment 분석에서는 "완전히 대체 가능까지는 아닌데…" 같은 부정 문맥이 긍정으로 잘못 카운트되는 거짓 양성 문제도 발견.

### v3에서 드러난 것 — Nemotron 페르소나는 "행동력 있는 저장러"가 거의 없다

120명 중 **단 한 명도 "저장 잘 함 → 대부분 방문" 답하지 않음**. 합성 페르소나의 자연 분포는 PDF에서 1순위로 잡았던 "수집형(저장 잘 함 → 안 감)"으로 자동 수렴.

→ **실제 사용자 모집할 때는 행동 기반 사전 스크리닝 필수**. "최근 한 달 안에 인스타 저장 장소 실제 방문 경험" 같은 질문으로 사전 필터링.

## 🎯 실제 사용자 인터뷰로 풀어야 할 1순위 질문들

시뮬로는 정확히 검증할 수 없거나 거짓 시그널이 섞이는 영역들 — 이걸 실제 인터뷰에 그대로 들고 가야 합니다.

| 순위 | 질문 | 왜 시뮬로 안 되나 |
|---|---|---|
| 1 | **"기존 인스타 저장함·메모장 대신 이걸 진짜로 쓸 것인가"** | Q19 카운트 거짓 양성, 시뮬은 "예의로 OK" 경향 |
| 2 | **월 X원이면 결제할 것인가, 어떤 조건이면 결제** | 데이터셋에 소득·자산 정보 없음 |
| 3 | **친구가 보낸 초대 링크로 가입한다면 어떤 진입 경험을 기대하는가** | 즉흥형 진입 시나리오, 시뮬은 1인칭이라 친구 관계 시뮬 불가 |
| 4 | **그룹 기능을 왜 그렇게 강하게 원하는가 (마케팅 직군)** | 직군 평균은 잡혔지만 동기·맥락은 정성 추출 필요 |
| 5 | **저장-방문 사이의 의사결정 흐름** | 시뮬은 행동을 그럴듯하게 지어내는 경향 |
